This is a course project for Applied Machine Learning. I collaborated with another student and trained models using various machine learning and deep learning techniques. This project was inspired by the idea of assigning helpfulness scores to the responses produced by LLMs. Primarily, the scoring was done manually and assigned helpfulness scores of 0 to 4 (least helpful to most helpful) for each response. However, this project was focused on automating that process by using trained models to predict the helpfulness scores. A link to the poster can be found here.

This is a project for Data Mining course. The objective of this project was to understand the relationship between various types of cancers and Area Deprivation Index (ADI) in Metropolitan Statistical Areas (MSA). I used multiple datasets - a cancer dataset, an ADI dataset, and two geospatial datasets to perform analyses and analyze the relationships between cancers and ADI in MSA. The preprocessing involved data cleaning in order to remove the non-relevant information, change data types, and drop NAN values. Then, I performed geospatial joins in order to join the cancer and ADI datasets on census block groups. The methods I used include dimensionality reduction using Principal Component Analysis (PCA), correlation analysis, cluster analysis using KMeans clustering, and topic modeling using Singular Value Decomposition (SVD). A link to the presentation can be found here.

This is a final course project for Networks course. I collaborated with three other students to investigate how actor and crew success in film industry is determined based on their collaboration patterns within the industry. This project utilized the TMDB 5000 Movies Datasets from Kaggle that allowed the team to incorporate rich information on cast, crew, genres, revenues, and budgets. The methods used were Graph Convolutional Networks (GCNs), Community Detection, and Centrality Measures (such as eigenvector centrality and degree centrality). The presentation link can be found here.

This project is for the Object Oriented Programming course (SI 206). A team of three students worked on museum data. Each person got involved in a separate museum Application Programming Interface (API). I worked with the Harvard Art Museum API and extracted data related to artists and their artworks. The goal of the project was to compare the data for each museum to identify the average duration to complete the artworks. This project involved making api calls to museum api to retrieve information, cleaning the data, creating database and database tables, storing data into database tables, and creating visualizations. A link to the GitHub repository with code can be found here.

This project was for a data manipulation course. I worked with real-world datasets from Southeast Michigan with a team of four.
